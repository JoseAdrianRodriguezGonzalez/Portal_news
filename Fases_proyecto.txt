	Se ha definido la arquitectura de persistencia de datos híbrida utilizando tres motores de base de datos distintos, asignando a cada uno la responsabilidad que mejor se adapta a su naturaleza.


	MongoDB (Gestión de Contenido - Noticias):

	Almacenamiento principal de las noticias y artículos.

	guardar estructuras de datos. para el contenido de las noticias (títulos, cuerpo del artículo con formato, metadatos, etiquetas y categorías) que pueden variar en estructura o tamaño.

	Implementación: Modelo Noticia definido en Mongoose.

	Apache Cassandra (Gestión de Usuarios y Auditoría):

	Rol: Almacenamiento de perfiles de usuario (Reporteros/Admin) y registro de actividad (Bitácora).

	Base de datos columnar optimizada para alta disponibilidad y escritura masiva. para la tabla de bitacora, donde la escritura de logs de eventos es constante y requiere rapidez.

	Implementación: Tablas usuarios y bitacora en el keyspace portal_news.

	Redis (Caché y Rendimiento):

	Rol: Gestión de datos volátiles de acceso rápido.

	Almacenamiento en memoria de clave-valor. Se utilizará para almacenar tokens de sesión (JWT) y cachear las noticias más leídas o recientes, reduciendo la carga de lectura directa sobre MongoDB y acelerando la respuesta del frontend.

	Implementación: Servicio redis conectado y cliente configurado en el backend.
